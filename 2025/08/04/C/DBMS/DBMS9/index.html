📘 从零构建数据库系统（五）：执行计划与查询优化器设计

在数据库系统中，查询优化器（Query Optimizer）是负责将用户的 SQL 查询转换为高效执行计划的核心组件。本篇将解析我在 DBMS 项目中如何设计和实现一个简单但结构清晰的 查询优化器（Planner） 模块。

🧠 1. 查询优化器的作用

优化器的目标是：在多种可选的查询方式中，选择一个代价最小、执行速度最快的方案。比如以下 SQL：

可能有多种执行策略：

  • 顺序扫描（TableScan）
  • 索引扫描（IndexSelect）
  • 投影优化（Project）
  • 谓词下推（Select）

优化器负责对这些策略进行组合和封装。

🧩 3. 各类 Plan 实现模块

3.1 TablePlan（全表扫描）

它负责扫描整个表，生成迭代器。最基础的 Plan 类型。

3.2 SelectPlan（条件过滤）

它在现有 Plan 上增加过滤功能，相当于“谓词下推”。

3.3 ProjectPlan（字段投影）

它删除无关字段，返回指定字段列表，属于 IO 优化。

3.4 ProductPlan（笛卡尔积）

实现连接两个表时的基本方式（未使用索引连接或哈希连接）。

✨ 5. 实现亮点与拓展性

  • 使用函数指针封装子类行为，C 语言中实现“多态”

  • 每个 Plan 实现了估算代价（记录数、IO 代价）的方法，为未来成本模型提供基础

  • 可轻松扩展支持：

    • 索引选择（IndexSelectPlan）
    • 聚合（GroupByPlan）
    • 联接优化（JoinPlan）
